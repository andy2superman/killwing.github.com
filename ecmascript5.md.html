## 对象属性特性(Property Descriptor) Descriptor|Default|Comment -|-|- value|undefined|当读取属性时返回的值 get|undefined|属性获取方法 set|undefined|属性设置方法 configurable|false|属性是否可被删除，以及其特性是否可以改变 enumerable|false|属性是否可被`for...in`/`Object.keys()`枚举 writable|false|属性是否可以改变 ### 创建 `Object.create(proto, props)` 使用一个指定的原型对象和一个额外的属性对象创建一个新对象。 ### 定义 `Object.defineProperty(obj, prop, descriptor)` `Object.defineProperties(obj, descriptors)` ### 检索 `Object.getOwnPropertyDescriptor(obj, prop)` 返回属性特性对象。 `Object.getPrototypeOf(obj)` 返回对象的原型，等同于`Object.__proto__`。 `Object.keys(obj)` 返回一个字符串数组，包含对象自己的可枚举（enumerable）属性。 `Object.getOwnPropertyNames(obj, prop)` 同上，但还包含 enumerable 标志设置为 false 的属性。 定义，检索都只针对自己的属性，不包括原型链中的属性。 ### 锁定 `Object.preventExtensions(obj)` 阻止将任何新属性添加到对象。 `Object.isExtensible(obj)` 测试对象是否可扩展。 `Object.seal(obj)` 首先调用`Object.preventExtensions()`，阻止添加新属性，然后将所有对象特性的 configurable 标志设置为 false。 `Object.isSealed(obj)` 测试对象是否被封印。 `Object.freeze(obj)` 调用`Object.seal()`来封印对象， 然后将对象特性的 writeable 标志设置为 false。 `Object.isFrozen(obj)` 测试对象是否冻结。 静态方法中的锁定方法都不针对原型，且操作禁止时不会抛出错误除非使用严格模式，没有解锁方法。 ### Accessor 定义和检索getter/setter的实例方法(Mozilla扩展): `__defineGetter__(name, callback)` `__defineSetter__(name, callback)` `__lookupGetter__(name)` `__lookupSetter__(name)` gettter/setter的字面定义法： ``` obj = { _x: 1, get name() {return this._x}, set name(v) {this._x = v} } ``` ##
