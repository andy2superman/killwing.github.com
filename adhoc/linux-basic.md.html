## Memory Segment Layout ``` +-----------------------------+ max | | | kernel space | | | +-----------------------------+ | | +-----------------------------+ stack offset | | | | stack | | | v | +-----------------------------+ <- stack top | | | | +-----------------------------+ RLIMIT_STACK | | +-----------------------------+ mmap offset | | | | heap (mmap) | | | v | +-----------------------------+ | | | | | | | | | | +-----------------------------+ <- break (heap top) | | | ^ | | heap (brk) | | | | | | | +-----------------------------+ brk offset | | +-----------------------------+ | bss | +-----------------------------+ | data | +-----------------------------+ | text | +-----------------------------+ 0 ``` 为了安全防止hack，stack和heap(brk/mmap)的起始地址都有一个随机的offset。 编译时确定.text, .data, .bss, .stack的大小（静态分配）。 ### 栈 (.stack) 保存局部变量，函数参数。 调用函数时产生一个新的栈帧在栈顶，函数返回时销毁。数据按照FIFO的顺序存放。 栈的最大大小由`RLIMIT_STACK`决定，通常为8MB。 进程里的每个线程都有自己的栈。 栈映射的物理内存只增不减，栈收缩时并不释放。 ### 堆 (.heap) `malloc/new`调用系统调用`brk(sbrk)`/`mmap(mmap2)`动态分配内存。开始调用的时候，分配虚拟内存，并不占用实际的物理内存，只有这块内存被初始化或使用的时候才分配物理内存。 根据申请的内存大小和阈值`M_MMAP_THRESHOLD`，分为两部分： * 堆：当申请内存小于阀值时，调用`brk`。 * 内存映射段：当申请内存大于等于阀值时，调用`mmap`。（匿名映射，此段还保存用文件映射加载的动态库） 阀值默认为128kB，而且可以动态调整：当分配的内存大于当前的阈值，且小于等于`DEFAULT_MMAP_THRESHOLD_MAX`(默认512kB)时，阈值就被设为当前分配的值，而且`M_TRIM_THRESHOLD`被设为2倍于当前分配的值。如果`M_TRIM_THRESHOLD`, `M_TOP_PAD`, `M_MMAP_THRESHOLD`, `M_MMAP_MAX`任意一个被显式设置，则动态调整被禁止。 `free/delete` 调用系统调用`brk`/`munmap`释放内存。 通过`mmap`分配的，立即释放回OS。 通过`brk`分配的，并不立即释放，而是积累到阈值`M_TRIM_THRESHOLD`时才释放。如果这块内存不在堆顶也不能释放，必须等到堆顶内存先释放。 ### 非初始化数据段 (.bss) 未赋初值的全局变量，局部及非局部静态变量，初始化为0。 内存区域（读写）并不映射到二进制可执行文件中。 ### 初始化数据段 (.data) 已初始化的全局变量，局部及非局部静态变量。 内存区域（读写）映射到二进制可执行文件中，但更新并不会反映到文件。 ### 正文段 (.text) 可执行代码段，字符串常量。 内存区域（只读）映射到二进制可执行文件中。 只有.text及.data和二进制可执行文件的大小有关。 ## Loop Device Loop Device设备指映射正常的文件而形成的虚拟块设备。 生成初始化为0的1G文件：(自动稀疏？) `dd if=/dev/zero of=vdisk bs=1M count=1000` 使用[稀疏文件](http://en.wikipedia.org/wiki/Sparse_file)： `dd if=/dev/zero of=vdisk bs=1M seek=1000 count=0` `du -h vdisk` 查看实际占用空间。 格式化： `mkfs.ext4 vdisk` 挂载： `mount vdisk vdir` mount会自动关联可用的loop device（umount自动取消关联），`losetup -a`查看。 指定关联：`mount vdisk vdir -o loop=/dev/loop0` 手动关联：`losetup /dev/loop0 vdisk` 取消关联：`losetup -d /dev/loop0` 默认最多使用8个loop device，改上限`/etc/modprobe.conf`里的`options loop max_loop=15`，使生效`modprobe -v loop`。 ## Mounting ### 自动挂载 [/etc/fstab](http://man7.org/linux/man-pages/man5/fstab.5.html) 格式 `设备名 挂载目录 文件系统 挂载参数 备份标志 自检顺序` `//server001/share$ /root/remote cifs defaults,user=kelvin,password=123456 0 0` 挂载参数: defaults, ro, rw, noexec, nodev, nosuid, nosgid, nouser 备份标志: 0为不备份，1为要备份，一般根分区要备份 自检顺序: 0为不自检，1或者2为要自检，如果是根分区要设为1，其他分区只能是2 ### 挂载传播 只有新挂载才被隔离，原有挂载点（如tmpfs, proc）继承。 mount propagation: 挂载传播，挂载点属性 * shared (rshared 递归): 双向传播 * slave (rslave 递归): 单向从 source 传播到 target * private: 不传播 * unbindable: 不可绑定挂载 更新 `mount --make-shared {mount-point}` 也可以在挂载时指定 查看 `findmnt -o TARGET,PROPAGATION {mount-point}` 当一个新 mount ns 创建时，复制所有存在的挂载点(vfsmount)。用 `unshare -m --propagation unchanged sh` 可以保持原挂载点属性不变，默认全变为 private 即全部相互隔离。 一般挂载：创建 vfsmount, 其 `mnt_root` (dentry) 为创建的设备目录信息。 绑定挂载：创建 vfsmount，其 `mnt_root` (dentry) 指向原目录。 [Shared subtrees](https://lwn.net/Articles/159077/) [Mount namespaces and shared subtrees](https://lwn.net/Articles/689856/) [Mount namespaces, mount propagation, and unbindable mounts](https://lwn.net/Articles/690679/) ## 进程 ### 进程组 进程组和作业控制有关，一个作业的所有进程属于同一个进程组。 一个进程只能为它自己或子进程设置 PGID。一般一个进程和其子进程属于同一个进程组。 每个进程组有个组长进程，其 PGID 等于 PID。只要有一个进程存在，进程组就存在，与组长是否存在无关。 ### 会话 会话是一个或多个进程组的集合。 会话首进程是创建该会话的进程，同时也是新进程组的组长进程。 如果会话有一个控制终端，则有一个前台进程组（最多一个），其他为后台进程组。中断键产生的信号（或会话首进程退出的挂断信号）发送给前台进程组的所有进程。 一般一个 shell 登录产生一个新的会话，shell 进程为会话首进程。 ## 信号 新 pid ns 中 pid 为1的进程，没有被加上默认信号处理（不区分 ns），在当前 ns 或其他 ns 也不能被 sigkill，但在父 ns 可以，当其退出时，pid ns 里面的所有进程收到 sigkill 退出。
