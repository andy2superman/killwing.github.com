## keepalived 利用 VRRP (虚拟路由冗余协议) 地址漂移，实现主备切换。 安装：`apt install keepalived` 配置：`/etc/keepalived/keepalived.conf`: ``` vrrp_script chk_haproxy { script "killall -0 haproxy && exit 0 || exit 1" weight -10 # 失败时减少的优先级 interval 3 # 每3秒检查一次 fall 3 # 连续3次认为失败 rise 1 # 连续1次认为成功 } vrrp_instance HA { state MASTER # 初始状态 MASTER/BACKUP interface bond0 virtual_router_id 111 priority 101 advert_int 1 authentication { auth_type PASS auth_pass 1234 } virtual_ipaddress { 10.18.21.111/24 } track_script { chk_haproxy } } ``` * `vrrp_script` 用来监控业务进程。如果不配置，则只能监控网络故障和 keepalived 本身。 * `virtual_ipaddress` VIP 配置为可用的同网段 IP。 * `virtual_router_id` 组标识，一般为 VIP 最后一段。 * `priority` 初始 master 节点上为 101，其余两个上为 100。 * 可以直接配置 `virtual_server` / `real_server` 来支持 ipvs。 ``` virtual_server 10.18.21.111 8080 { delay_loop 6 # 每6秒检查一次real_server状态 lb_algo rr # LVS算法 lb_kind DR # LVS模式 protocol TCP real_server 10.18.21.101 8080 { weight 3 TCP_CHECK { connect_timeout 30 # 连接超时时间 nb_get_retry 3 # 重连次数 delay_before_retry 3 # 重连间隔时间 connect_port 8080 # 健康检查端口 } } real_server 10.18.21.102 8080 { weight 3 TCP_CHECK { connect_timeout 30 nb_get_retry 3 delay_before_retry 3 connect_port 8080 } } } ``` ## haproxy 可以进行七层/四层负责均衡。 利用客户端 IP，cookie, session 实现保持会话。 安装：`apt install haproxy` 配置：`/etc/haproxy/haproxy.cfg`: 在默认配置基础上增加： ``` frontend apiserver-fe bind *:7443 default_backend apiserver-be mode tcp option tcplog backend apiserver-be mode tcp option tcplog option tcp-check balance roundrobin default-server inter 10s downinter 5s rise 2 fall 2 slowstart 60s maxconn 250 maxqueue 256 weight 100 server apiserver-master1 10.18.21.1:6443 check server apiserver-master2 10.18.21.2:6443 check server apiserver-master3 10.18.21.3:6443 check ``` * frontend 中 bind 为 LB 监听端口。 * backend 中 master 节点 IP 修改为对应地址。 * `defaults.timeout client/server` idle 超时建议调整为一个小时。 * 转发模式类似 lvs 的 FULLNAT，HTTP 可以加 `X-Forwarded-For` 带上真实客户端 IP。 ## lvs/ipvs 四层负责均衡。 四种转发模式： DR (Direct Routing) (-g) 对于收到的请求包，只改写请求包的MAC地址为后端RS的MAC地址；对于发出的响应包，直接发给客户端。 RS必须利用回环网卡绑定监听 lvs 的 VIP 以此接收和发送数据包。 没有 NAT，不能端口映射。 NAT (-m) 对于收到的请求包，进行DNAT；对于发出的响应包，进行SNAT。 RS必须修改默认网关为 lvs 的 VIP，让发出的包进行 SNAT，否则客户端不认识内网地址。 FULLNAT (-b) 对请求包和响应包都做SNAT+DNAT。 TUN (IP Tunneling) (-i) 对于收到的请求包，将其封装在一个新的 IP 包中发给RS；对于发出的响应包，进行SNAT。 RS收到IP Tunnel包，必须进行解包拿到原始 IP 包进行处理。 RS必须修改默认网关为 lvs 的 VIP，让发出的包进行 SNAT，否则客户端不认识内网地址。 LB 算法 * 轮询（Round Robin） * 加权轮询（Weighted Round Robin） * 最小连接数（Least Connections） * 加权最小连接数（Weighted Least Connections） * 地址哈希（Address Hash） 安装：内核默认支持 `lsmod|grep ip_vs`, `modprobe ip_vs` 配置工具：ipvsadm ``` # 添加VS，VIP 192.168.1.100，rr轮询算法 ipvsadm -A -t 192.168.1.100:80 -s rr # 添加RS，DR模式，权重为2 ipvsadm -a -t 192.168.1.100:80 -r 192.168.1.123 -g -w 2 ipvsadm -a -t 192.168.1.100:80 -r 192.168.1.124 -g -w 2 ipvsadm -a -t 192.168.1.100:80 -r 192.168.1.125 -g -w 2 ```
