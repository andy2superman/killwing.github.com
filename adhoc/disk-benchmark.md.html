## Buffer & Cache ### 查看 `free -m` `cat /proc/meminfo` ``` total used free shared buffers cached Mem: 1002 769 232 0 62 421 -/+ buffers/cache: 286 715 Swap: 1153 0 1153 ``` (-buffers/cache) used内存数：286M (指的第一部分Mem行中的used - buffers - cached) 反映的是被程序实实在在用掉的内存（不包括系统缓存） (+buffers/cache) free内存数: 715M (指的第一部分Mem行中的free + buffers + cached) 反映的是可以挪用的内存总数（包括系统缓存） swap: 交换分区 Page Cache (cached):，针对磁盘块的读写 Buffer Cache (buffers): 针对文件dentries和inode的读写 ### 清除 `echo 1/2/3 > /proc/sys/vm/drop_caches` 1: page cache 2: buffer cache 3: both 还未真正写入磁盘的脏数据不会被清除，可先用`sync`强制同步数据到磁盘上以释放更多空间。 ### 配置 `/proc/sys/vm/` * dirty_background_bytes 脏数据达到多少字节时，后台刷新线程开始写回脏数据到磁盘 * dirty_background_ratio 脏数据占总可用内存比例多少时，后台刷新线程开始写回脏数据到磁盘 * dirty_bytes 脏数据达到多少字节时，进程自己开始写回脏数据到磁盘 * dirty_ratio 脏数据占总可用内存比例多少时，进程自己开始写回脏数据到磁盘 * dirty_expire_centisecs 脏数据在cached里的过期时间，过期的数据需要被刷回磁盘，单位百分之一秒 * dirty_writeback_centisecs 刷新同步线程的唤醒间隔，单位百分之一秒 脏数据指在cached中尚未同步到磁盘的数据。同步时所有写操作都会阻塞直至同步完成。 ### 监控 `watch grep ^Cached /proc/meminfo` Page Cache大小 `watch grep -A 1 dirty /proc/vmstat` 脏页写回活动 `watch cat /proc/sys/vm/nr_pdflush_threads` 活动的pdflush线程 ## Benchmark ### read 从磁盘读：`hdparm -t /dev/sda1` 从缓存读：`hdparm -T /dev/sda1` ### write 写入256M数据并最后同步到磁盘：`dd bs=1M count=256 if=/dev/zero of=test conv=fdatasync` 参数`oflag=dsync`表示每次写操作都同步一次，不加参数则由系统自动同步。 ## Reference http://www.ibm.com/developerworks/linux/library/l-linux-filesystem/ http://www.thesubodh.com/2013/07/what-are-exactly-odirect-osync-flags.html https://www.kernel.org/doc/Documentation/sysctl/vm.txt https://romanrm.net/dd-benchmark
